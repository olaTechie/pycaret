# Public-Health Data Analyst Review — mltoolkit-plugin

**Reviewer lens:** applied epi / clinical-prediction fit, ethics and equity, TRIPOD+AI / STARD / CONSORT-AI / PROBAST alignment, paper-ready outputs.
**Date:** 2026-04-14
**Scope:** `mltoolkit-plugin/` at HEAD of `master` (6f40561f).

---

## 1. Summary

mltoolkit-plugin delivers a clean, sklearn-native four-task pipeline with a serviceable packaging tier. For a generic ML demo it is tight; for a **public-health / clinical-prediction audience it is not yet usable without heavy user rewrites.** Core gaps across every supervised skill: (a) no group-fairness or equity audit surface — protected attributes are silently consumed as predictors (`skills/classify/references/preprocessing.py:31` `build_preprocessor`, `skills/regress/references/preprocessing.py:29`); (b) no calibration assessment, bootstrap CIs, or decision-curve analysis, so TRIPOD+AI Items 13b/15/16 cannot be reported from plugin outputs (`skills/classify/references/classify_reference.py:143` `evaluate`); (c) missing-data handling is hard-coded median/most_frequent imputation with no MAR/MCAR/MNAR surfacing or multiple imputation path (`skills/classify/references/preprocessing.py:39`); (d) no Table 1 / subgroup / stratified reporting; (e) no dataset-provenance, consent, or data-sheet prompt in `skills/setup/SKILL.md`. The plugin is also weaker than PyCaret on the fairness/interpretability axis (no `check_fairness`, no `interpret_model`/SHAP, no `calibrate_model` — see capability reference §1.9, §1.15, §1.6). I recommend P0 work on a fairness helper and a calibration/CI helper before any clinical user is asked to generate a paper-ready pipeline.

## 2. Current-spec gaps

Gaps present in the plugin *as specified today* (i.e., they would still exist even if the plugin had no PyCaret target to close against).

- **No group-fairness reporting anywhere.** There is no code path that accepts a `--sensitive-feature` or `--group-col` argument, no per-group metric table, no disparity summary. Evidence: `skills/classify/SKILL.md:44-51` (adaptation rules mention imbalance + SMOTE but never protected attributes); `skills/classify/references/classify_reference.py:143-183` (`evaluate` computes a global classification report only); `skills/regress/references/regress_reference.py:109-155` (`evaluate` computes global R²/RMSE/MAE only). A public-health user cannot answer "does my model under-perform for subgroup X?" without rewriting the script. In contrast PyCaret ships `check_fairness` (capability reference §1.15, §6.1).
- **No calibration assessment for classifiers.** `evaluate` in `skills/classify/references/classify_reference.py:143-183` produces confusion matrix, ROC, PR, and permutation importance — but no calibration plot (reliability diagram), no Brier score, no expected calibration error, and no `CalibratedClassifierCV` wrapping. TRIPOD+AI 15/16 (calibration intercept, slope, calibration-in-the-large) is unreportable.
- **No confidence intervals on any metric.** All metrics are point estimates. `compare_models` reports `*_std` across folds (`skills/classify/references/classify_reference.py:101-104`) but no BCa / percentile bootstrap CI on holdout metrics. TRIPOD+AI 13b requires 95% CIs on performance measures.
- **Class imbalance handled only as a hand-wave.** `skills/classify/SKILL.md:50` says "If a class imbalance >4:1 is detected in EDA and imblearn is installed, mention SMOTE as an option" — but the EDA stage never detects or flags imbalance numerically. `classify_reference.run_eda` (lines 56-81) produces a bar chart with no prevalence statistic. Rare-outcome contexts (common in PH: sepsis, SAE, rare disease) get no special handling.
- **Split is a random holdout with no temporal / site awareness.** `skills/classify/references/classify_reference.py:203-207` uses `train_test_split(random_state=42)`. For clinical data this leaks across sites/time and overstates performance. No `GroupKFold`, no `StratifiedGroupKFold`, no `TimeSeriesSplit`, no external-validation stub.
- **Missing-data handling is opaque.** `skills/classify/references/preprocessing.py:39,47,59` and `skills/regress/references/preprocessing.py:33,38,48` hard-code `SimpleImputer(strategy="median"|"most_frequent")`. No missingness pattern report, no MAR/MCAR/MNAR discussion in `SKILL.md`, no multiple-imputation option, no missing-indicator columns. `setup_reference.py:35-42` does compute per-column missing counts but never flags patterns (e.g., MNAR indicators, visit-pair missingness, >30% missing).
- **Threshold is fixed at 0.5.** No `optimize_threshold` surface, no cost-weighted threshold selection, no net-benefit / Youden-J tuning. For screening vs. triage paper comparisons this is load-bearing.
- **Clustering/anomaly skills use `select_dtypes(include="number")` unconditionally.** `skills/cluster/references/cluster_reference.py:31-37` and `skills/anomaly/references/anomaly_reference.py:29-35` silently drop all categorical predictors — including indicator variables for race/ethnicity/insurance that PH users often encode numerically as 0/1 but label categorically. `SKILL.md` promises to "prompt the user" (`skills/cluster/SKILL.md:39`) but the reference code offers no `--one-hot` or `--include-categorical` flag, so the skill's instructions cannot be honored without editing.
- **Anomaly contamination is a hyperparameter, not an estimate.** `skills/anomaly/SKILL.md:38` tells the user to "ask user for expected contamination rate (default 5%)" but never surfaces the tension between a prior-based `contamination` and a tail-based `score_samples` threshold. No mention that LOF/IForest anomaly rates are fraction-of-training artifacts rather than out-of-sample rates.
- **Cluster refit path may diverge from the scored labels.** `skills/cluster/references/cluster_reference.py:120-121` creates a *fresh* estimator and refits it to serialize, but `joblib.dump` then saves that fresh instance even though the `_fit_labels` result is discarded. Reloaded model may produce different labels than the ones saved to `assigned.csv` (esp. DBSCAN ordering, GMM random state variation without fixed seed on re-fit path). For a cohort-assignment study this is reproducibility-critical.
- **No "dataset card" / consent / provenance prompt.** `skills/setup/SKILL.md:17-32` has no step that asks about consent status, IRB approval, dataset licence, or recipe for `datasheet.md`. PH-ethics QC requires at minimum a provenance statement.
- **No seed/version capture.** Session artifacts don't record sklearn/pandas/numpy/xgboost versions, OS, git hash, or CPU. TRIPOD+AI 7a/19b requires reproducibility metadata. `joblib.dump(best_model, out/"model.joblib")` (e.g., `classify_reference.py:227`) is the only reproducibility artifact.
- **No supplementary-appendix scaffold.** Even Tier C (`skills/package/references/tier_c_transform.py`) emits `src/preprocess.py`, `src/train.py`, `src/predict.py`, `tests/`, optional `api.py`, `Dockerfile` — but no `reports/supplement/` with Table 1, subgroup metrics, calibration plot, decision-curve, or a methods-section scaffold.

## 3. PyCaret-target gaps

Capabilities present in PyCaret (per `docs/superpowers/reviews/2026-04-14-pycaret-capability-reference.md`) that a public-health user is highly likely to miss in mltoolkit.

- **`check_fairness(estimator, sensitive_features)` missing** (capability reference §1.15, §6.1). mltoolkit has no equivalent in any skill. For TRIPOD+AI 13c / equity reporting this is the single largest PyCaret→mltoolkit gap for PH audiences.
- **`calibrate_model` missing** (capability reference §1.6). No isotonic/sigmoid calibration wrapper; no reliability diagram via `plot_model(plot='calibration')`.
- **`interpret_model` (SHAP) missing** (capability reference §1.9). Permutation importance in `classify_reference.evaluate` (lines 167-181) is a partial substitute but does not give per-observation explanations — required for "which patient-level features drove this prediction?" and clinician-review narratives.
- **`optimize_threshold` missing** (capability reference §2.5). No threshold-moving surface.
- **`predict_model(..., data=new)` style unified predict missing.** Packaged Tier A/B/C emits a model and a README snippet (`skills/package/references/tier_b_readme_template.md:34-39`) saying `model.predict(new_data)` — but there is no wrapper that appends `prediction`, `prediction_score`, and (for anomaly) `Anomaly`/`Anomaly_Score` columns as PyCaret does.
- **`dashboard(estimator)` missing** (capability reference §1.16). No explainer-dashboard-equivalent. For a clinician-user handoff this is a meaningful reporting gap.
- **`check_drift` missing** (capability reference §2.5). No post-deployment PH-relevant drift surface.
- **Experiment tracking missing** (capability reference §1.18 / §6.3). No MLflow / W&B / Comet / DagsHub integration. `best_params.json` + `holdout_metrics.json` + `leaderboard.csv` is the full tracking surface today.
- **`get_metrics` / `add_metric` / `remove_metric` missing** (capability reference §1.17). A PH user cannot plug in PPV@fixed-recall, NNT, NRI, IDI, or decision-curve net-benefit without editing `classify_reference.py` directly.
- **Model-zoo narrower than PyCaret on classification/regression**:
  - Classification: no `lda`, `qda`, `rbfsvm`, `gpc`, `dummy` (baseline) — see `skills/classify/references/model_zoo.py:22-110` vs. capability reference §2.1. `dummy` baseline absence is the PH-important one (need a "prevalence-only" floor for claim framing).
  - Regression: no `huber`, `ransac`, `tr` (Theil-Sen), `br`, `ard`, `par`, `llar`, `lar`, `omp`, `kr`, `dummy` — see `skills/regress/references/model_zoo.py:21-69` vs. capability reference §3.1. Robust regressors matter for PH data (skewed costs/utilisations).
- **Anomaly zoo much narrower**: mltoolkit has `iforest`, `lof`, `elliptic`, `ocsvm` (`skills/anomaly/references/model_zoo.py:9-14`). PyCaret covers `abod`, `cluster`, `cof`, `histogram`, `knn`, `pca`, `mcd`, `sod`, `sos` additionally (capability reference §5.1). For rare-event PH use (billing-anomaly, surveillance) the HBOS/ABOD families are commonly cited.
- **No `finalize_model` step** (capability reference §1.11). TRIPOD+AI encourages retraining on the combined train+holdout before deployment; mltoolkit never does this and never tells the user to.

## 4. Paper-reporting gaps

What a PH analyst would need to pull out of `.mltoolkit/results/` and paste into a manuscript — and cannot, today.

- **Methods-section scaffold is absent.** No file like `methods_snippet.md` that summarises: split strategy, preprocessing pipeline, model-zoo tried, CV folds, tuning budget, primary metric, software versions. Would fit naturally in `skills/package/references/tier_b_readme_template.md:1-39` but is not generated.
- **Table 1 (baseline characteristics) not generated.** No `results/table1.csv` with overall + by-outcome means/proportions and standardised mean differences. Grep-confirmed absent: no `table1`, `tableone`, `SMD` anywhere in the plugin.
- **STROBE / TRIPOD flow diagram not generated.** No figure describing inclusion/exclusion, train/test counts, class counts, missing-data drop rates.
- **No per-class / per-group performance table.** `classification_report.csv` (`classify_reference.py:147-148`) gives per-class precision/recall/F1 but stops there. No stratified metrics by a user-supplied grouping variable.
- **No calibration plot / Brier / ECE.** Required for TRIPOD+AI 15.
- **No decision-curve / net-benefit plot.** Required for clinical-utility framing; typically expected in any prognostic-model paper. Not present.
- **No bootstrap CI on holdout metrics.** `holdout_metrics.json` (`classify_reference.py:117-119` etc.) is point-estimate only.
- **No Q-Q / calibration-in-the-small / Hosmer-Lemeshow for classifiers.** Regression has Q-Q (`regress_reference.py:136-140`) — classification has no analog.
- **No reproducibility appendix** with library versions, random seeds, CV splits, hardware. Only `best_params.json` and `model.joblib` are exported.
- **Figures saved only as PNG+PDF** (`test_shared.py:42-45`). That's actually fine for papers, but no 300-dpi / journal-size variant is emitted, no typography set for print. `plotting.set_style()` is called but its contents are not visible in this review; depending on DPI it may fail journal submission.
- **No ready-to-submit STARD/TRIPOD/CONSORT-AI checklist file.** Tier C should arguably emit a `reports/tripod_ai_checklist.md` stub.

## 5. Ethics / bias / equity audit

This section is first-class, not bundled.

- **Protected attributes are consumed silently as predictors.** `skills/classify/references/preprocessing.py:23-28` selects every categorical as either low-card OHE or high-card TargetEncoder. A column named `race` or `ethnicity` flows in with no prompt, no dedicated handling, and no audit. Evidence: no grep hit for `sensitive`, `protected`, `race`, `ethnicity` anywhere outside docs. Severity: P0.
- **TargetEncoder leaks outcome into high-cardinality protected-like columns.** `preprocessing.py:53` — `TargetEncoder` fit-transforms the full training set against y. If `zip_code` or `county_fips` is high-cardinality, this silently encodes neighbourhood-level outcome into the model. Known proxy-discrimination vector.
- **No equity-by-design prompt in setup.** `skills/setup/SKILL.md:20-37` infers task type but never asks about protected attributes, consent, or disparate-impact concern. TRIPOD+AI 2 (study objectives / target population) cannot be answered.
- **Subgroup performance never computed.** See §4. A model can be "90% AUC" overall and 60% in a minority subgroup — mltoolkit cannot surface this.
- **Representational-harm audit absent.** No check on sample-size-by-subgroup, no warning when a subgroup has <N records.
- **Dataset provenance / consent signal absent.** No `datasheet.md` scaffold, no prompt for IRB number, no licence capture in `.mltoolkit/`.
- **Anomaly detection ethics untreated.** `skills/anomaly/SKILL.md` has no warning about using anomaly scores against protected groups (false-positive fraud / welfare-abuse flagging is an active policy-failure pattern). Evidence: no grep hit for "bias", "fairness", or "protected" in that file.
- **Clustering ethics untreated.** `skills/cluster/SKILL.md` contains no warning that cluster labels become de-facto strata — when reused for resource allocation they reify historical inequities. No prompt.
- **No re-identification risk note in packaging.** Tier A/B/C emit the trained pipeline; for image / small-N tabular data the model can memorise. `tier_b_readme_template.md` should at minimum flag this; it does not.
- **No "harm statement" in package README.** Contrast: modern model-card conventions (e.g., Hugging Face, Google Model Cards) prescribe an intended-use + out-of-scope section. `tier_b_readme_template.md:1-39` has neither.

## 6. Prioritized recommendations

**P0 (ship before PH users are pointed at this):**

1. **Add a fairness helper.** Implement `references/_shared/fairness.py` exposing `group_metrics(y_true, y_pred, y_proba, groups)` returning per-group AUC, PR-AUC, PPV, NPV, sensitivity, specificity, FPR, FNR, calibration, and disparities (ratio and difference) vs. a reference group. Wire a `--group-col` flag into `classify_reference.py` and `regress_reference.py`. Emit `results/fairness_report.csv` + bar-plot figure. Closes both §5 and the `check_fairness` gap in §3.
2. **Add a calibration + bootstrap-CI helper.** Implement `references/_shared/calibration.py` with reliability diagram, Brier, ECE, calibration intercept/slope, plus `bootstrap_ci(metric_fn, y_true, y_pred, n_boot=1000)` for every holdout metric. Emit `results/calibration.json`, `artifacts/calibration_curve.png`, `results/holdout_metrics_ci.json`. Closes TRIPOD+AI 13b/15/16 gaps in §4.
3. **Add group-aware split option.** Accept `--group-col` for `GroupKFold`/`StratifiedGroupKFold`, and `--time-col` for `TimeSeriesSplit`. Update `skills/classify/SKILL.md` and `skills/regress/SKILL.md` to document.
4. **Add a setup ethics prompt.** Extend `skills/setup/SKILL.md` to require (a) IRB / consent status, (b) protected-attribute list, (c) dataset provenance. Emit `.mltoolkit/datasheet.md` stub.

**P1 (ship before "paper-ready" claim):**

5. **Emit Table 1.** Add `references/_shared/table1.py` generating per-covariate mean±SD / N (%) overall and by outcome, with SMD. Called from `classify_reference.run_eda` and `regress_reference.run_eda`.
6. **Emit methods-section scaffold.** Tier B/C packaging should generate `reports/methods.md` prefilled with split strategy, preprocessing, zoo tried, CV folds, tuning budget, seeds, versions.
7. **Emit TRIPOD+AI / STARD / CONSORT-AI checklist stubs.** Tier B/C adds `reports/tripod_ai_checklist.md` with every item and the plugin's auto-populated rows where possible.
8. **Emit decision-curve plot** for classification via a `--decision-curve` flag.
9. **Add `calibrate_model` equivalent.** Wrap the tuned classifier in `CalibratedClassifierCV` on request; include in zoo comparison when `--calibrate` passed.
10. **Add `DummyClassifier` / `DummyRegressor` baselines** to both zoos. PH papers need a prevalence floor.
11. **Warn on imbalance + low outcome count numerically.** Extend `run_eda` to compute minority prevalence and events-per-variable, warn if EPV < 10.
12. **Expand missing-data handling.** Add `--imputation {median,iterative,knn,drop,multiple}` choice and a missingness-pattern figure.

**P2 (nice-to-have, ordered by PH relevance):**

13. Add SHAP / `interpret_model` equivalent (optional `shap` dependency; fall back to permutation importance).
14. Add `optimize_threshold` with cost/utility weights.
15. Add experiment-tracking hook (`--log-mlflow`, optional).
16. Add re-id / harm / intended-use sections to `tier_b_readme_template.md`.
17. Broaden zoos (robust regressors, HBOS/ABOD anomaly detectors, `lda`/`qda`).
18. Add `finalize_model` step: after holdout evaluation, retrain on full data and save as `model_final.joblib` alongside `model.joblib`.
19. Fix cluster serialization drift (`cluster_reference.py:118-127`).

## 7. Open questions

- Is the plugin intended to stay strictly sklearn-native, or is adding `fairlearn` / `tableone` / `shap` / `dcurves` as optional deps acceptable? (Maps onto P0/P1 scope.)
- Should fairness reporting be auto-on when any column name matches a protected-attribute dictionary, or strictly opt-in via `--group-col`? (Trade-off: autopilot vs. explicit consent.)
- Does the team intend to support temporal / multi-site splits, or is the "random 80/20" split the ceiling? (Affects whether `GroupKFold`/`TimeSeriesSplit` is P0 or P1.)
- Is Tier C expected to be clinical-deployment-grade (HIPAA / audit-log / encryption) or demo-grade? The Dockerfile presence suggests deployment intent; if so, re-id risk warnings and audit logging should be P0.
- Is PyCaret parity the explicit goal, or are "paper-ready PH outputs" the goal? If the latter, some PyCaret gaps (e.g., `dashboard`) deprioritise while TRIPOD+AI / Table 1 / decision-curve go up.
- Is there intent to add a time-series skill? (Capability reference §7.1.) PH surveillance tasks often need it.
- Should the plugin refuse to train on raw `race`/`ethnicity` without an explicit acknowledgement flag (akin to `--acknowledge-protected-attributes`)?
